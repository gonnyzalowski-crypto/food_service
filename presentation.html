Order Flow

Hardware Orders

1. Customer adds products to cart
2. Customer proceeds to checkout
3. Customer fills billing/shipping information
4. Order created with status: "Awaiting Payment"
5. Customer receives bank details
6. Customer makes bank transfer
7. Customer uploads payment receipt
8. Status changes to: "Payment Uploaded"
9. Admin reviews and confirms payment
10. Status changes to: "Payment Confirmed"
11. Admin creates shipment with tracking
12. Status changes to: "Shipped"
13. Customer receives tracking information
14. Status changes to: "Delivered" upon delivery

Software Orders

1. Customer adds software products to cart
2. Customer proceeds to checkout
3. Customer fills billing information (no shipping needed)
4. Order created with type: "software"
5. Customer receives bank details
6. Customer makes bank transfer
7. Customer uploads payment receipt
8. Admin reviews and confirms payment
9. Admin sends license key via email
10. Status changes to: "License Sent" / "Delivered"
11. Customer receives software instructions via email

Payment System

  • Supported method: Bank Transfer / Wire Payment
  • Bank details configurable in admin settings
  • Payment receipt upload: PDF/JPG/PNG, max 10MB
  • Admin manually verifies payment

Bank details include Bank Name, Account Holder, IBAN, BIC/SWIFT.

Notification System

Telegram

Email

  • Order confirmations
  • Payment confirmations
  • Shipment notifications with tracking
  • Software license delivery emails

Database Schema (Core Tables)

  • users: id, email, password_hash, full_name, phone, role, is_active, created_at
  • products: sku, name, slug, description, specs, category_id, pricing, stock, type (hardware/software)
  • categories: name, slug, description, image_url, parent_id, sort_order
  • orders: order_number, user_id, status, type (hardware/software/mixed), totals, addresses (JSON)
  • order_items: order_id, product_id, sku, name, quantity, unit_price, total_price
  • shipments: carrier, tracking_number, status, timestamps
  • payment_uploads: order_id, file_path, file_name, status, notes
  • tracking_communications: tracking_number, sender_type, message
  • settings: setting_key, setting_value

API Endpoints (Public)

  • GET / Home
  • GET /catalog Catalog (optional ?category={slug})
  • GET /product?sku={sku} Product detail
  • Cart: GET /cart, POST /cart/add, /cart/update, /cart/remove
  • Checkout: GET/POST /checkout
  • Tracking: GET/POST /track, GET /order/{id}
  • Payment upload: GET/POST /order/{id}/payment

API Endpoints (Admin + Utility)

Admin

  • GET /admin Dashboard
  • Orders: GET /admin/orders, GET /admin/orders/{id}, POST /admin/orders/{id}/status, POST /admin/orders/{id}/confirm-payment
  • Products: GET /admin/products, GET /admin/products/new, POST /admin/products, GET /admin/products/{id}/edit, POST /admin/products/{id}
  • Shipments: GET /admin/shipments, POST /admin/shipments
  • Settings: GET/POST /admin/settings
  • Customers: GET /admin/customers
  • Tickets: GET /admin/tickets

Utility
